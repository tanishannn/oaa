        <h2 id="sec1">一、前言</h2>

        <h3 id="sec1.1">1.1 目的</h3>

        <p>本文档旨在指导用户快速掌握观己OA系统的核心功能及操作流程，提高办公效率，规范工作流程。</p>

        <h3 id="sec1.2">1.2 适用范围</h3>

        <p>本说明书适用于公司所有员工，包括新入职员工及需要系统使用指导的现有员工。</p>

 

        <h2 id="sec2">二、系统概述</h2>

        <h3 id="sec2.1">2.1 系统简介</h3>

        <p>观己OA系统是集“会员预约管理、任务中心、档案管理、分期管理、财务管理”于一体的办公平台，支持电脑端（网页版）及移动端（公众号）访问，实现跨设备、跨场景的高效处理工作。</p>

        <h3 id="sec2.2">2.2 核心功能</h3>

        <ul>

            <li><strong>会员预约管理</strong>：统一管理与跟踪客户到店预约信息。</li>

            <li><strong>任务中心</strong>：集中管理各类待办任务、配置任务规则以及跟踪任务执行情况的统一入口。不同角色可根据权限查看和处理与自己相关的任务。</li>

            <li><strong>档案管理</strong>：规范化管理观己内部电子与纸质档案的核心功能模块。</li>

            <li><strong>分期管理</strong>：处理客户分期付款订单，精细化管理分期流程，控制回款风险。</li>

            <li><strong>财务管理</strong>：财务标准化流程线上处理（线下收款核款、滞纳金减免、推荐结清、退款等）。</li>

        </ul>

        <h2 id="sec4">四、核心功能详解</h2>

        <h3 id="sec4.1">4.1 会员预约管理</h3>

        <h4 id="sec4.1.1">4.1.1 创建会员</h4>

        <p>点击“会员中心”→“新增会员”→填写基本信息。</p>

        <h4 id="sec4.1.2">4.1.2 系统预约操作</h4>

        <p><strong>操作顺序：</strong>先建会员，再在预约面板约时间（分两步完成）。进入预约面板，新增预约→选择对应会员，填写预约表单（带*为必填项），完成提交后预约生效。</p>

        <div class="note">📌 注意：预约后不可直接取消，如需变更可在编辑选项中修改预约时间。</div>

        <h4 id="sec4.1.3">4.1.3 已到店（咨询助理操作节点）</h4>

        <p>顾客实际到店后，咨询助理需在系统中点击“已到店”确认。该动作作为服务流程的正式起点，触发后续咨询、接诊等环节，实现状态从“预约”到“在店”的系统流转。</p>

        <h4 id="sec4.1.4">4.1.4 会员资料审核</h4>

        <p><strong>功能解释：</strong>该模块为只读的审核结果通知看板，专供咨询助理查看审核驳回原因，并线下指导顾客修正资料，驱动“提交→审核→驳回→修正→重审”的闭环。</p>

        <p><strong>操作流程：</strong></p>

        <ol>

            <li><strong>顾客提交资料</strong>：咨询助理可在会员个人界面修改并提交该会员个人基本信息（身份证、直系亲属联系人姓名及电话、家庭住址等）。</li>

            <li><strong>后台人工审核</strong>：后台审核页面逐条核对资料，选择“通过”或“驳回”。若驳回，需填写具体驳回原因（如“身份证照片模糊”“手机号非本人”）。</li>

            <li><strong>结果同步展示</strong>：会员个人页面显示审核状态及驳回原因；通知看板同步更新，咨询助理可查看所有驳回记录。</li>

            <li><strong>重新介入</strong>：看到驳回后主动联系顾客，告知原因并指导修正。</li>

            <li>顾客修正后重新提交，系统再次进入审核队列，直至“审核通过”。</li>

        </ol>

 

        <h3 id="sec4.2">4.2 任务中心</h3>

        <h4 id="sec4.2.1">4.2.1 售后回访（售后模块）</h4>

        <p><strong>功能解释：</strong>针对已完成订单或售后服务后的客户进行问题跟进及结果记录，支持回访任务创建与完成状态追踪。售后回访侧重于问题解决与客户安抚。</p>

        <p><strong>创建回访任务：</strong>手动创建（在售后回访页面点击“新建回访任务”）或自动生成（订单完成后3天生成满意度回访；逾期未还款生成催收回访）。<br>

        <strong>执行回访：</strong>执行人通过多维度筛选查看待回访任务，通过电话/微信回访客户并记录结果，所有记录同步至会员详情。</p>

       

        <h4 id="sec4.2.2">4.2.2 市场回访（市场模块）</h4>

        <p><strong>功能解释：</strong>由市场人员执行，面向已完成牙贴面治疗的成交客户的周期性专项任务，聚焦裂变线索获取与转介绍引导。</p>

       

        <h4 id="sec4.2.3">4.2.3 病历管理（医生模块）</h4>

        <p>医生专属工作台，用于创建、维护和查阅患者电子病历，支持上传照片与结构化填写病历选项。详见“五、角色使用操作”中的医生SOP。</p>

       

        <h4 id="sec4.2.4">4.2.4 预约管理（售后模块）</h4>

        <p><strong>功能解释：</strong>提醒售后人员在当前治疗完成后，于指定时间节点为患者预约复诊时间。系统根据治疗计划自动触发提醒，确保复诊闭环。<br>

        <strong>操作流程：</strong>进入“任务中心-预约管理”，查看任务栏中的预约提醒列表，筛选需要提前联系的患者，通过电话/微信确认或调整复诊时间，并更新预约状态。</p>

       

        <h4 id="sec4.2.5">4.2.5 咨询管理（咨询模块）</h4>

        <p>面向咨询岗位，核心动作为每一位顾客填写详细的咨询记录，实现从初次接触、需求沟通节点记录。</p>

 

        <h3 id="sec4.3">4.3 订单管理</h3>

        <p>系统对所有订单类型进行统一创建、查询、跟踪、核销与结算的后台管理功能，涵盖以下三类订单：</p>

        <h4 id="sec4.3.1">4.3.1 全款订单</h4>

        <p>仅用于记录主营项目中的正价订单，即一次性支付全部费用，该类订单金额固定，支付即完成结算。</p>

        <h4 id="sec4.3.2">4.3.2 诊疗明细订单</h4>

        <p>记录每一次实际诊疗行为的详细清单（项目、操作医生、诊疗时间、服务人员等）。独立于订单类型，关联全款订单、主营活动订单或非主营订单。每次诊疗都必须生成诊疗明细订单，用于医疗记录。</p>

        <h4 id="sec4.3.3">4.3.3 业务订单列表</h4>

        <p>系统中展示所有业务订单的汇总界面，其中“业务订单”涵盖了诊所内发生的全部交易订单，不分主营或非主营，不分正价或活动，可进行个性化筛选。</p>

 

        <h3 id="sec4.4">4.4 售后管理</h3>

        <h4 id="sec4.4.1">4.4.1 回访任务配置</h4>

        <p><strong>功能解释：</strong>后台预设回访计划的管理模块。管理员可设定回访触发条件、执行时间、负责岗位、内容模板等，配置后系统自动生成并分配回访任务。</p>

        <div class="img-placeholder">🖥️ [示意图：回访任务配置页面]</div>

 

        <h4 id="sec4.4.2">4.4.2 贷后看板（贷后模块）</h4>

        <p><strong>功能解释：</strong>以卡片式指标、表格和对比数据为核心，实时掌握逾期风险、回款进度、催收任务执行情况以及个人表现，实现任务驱动、过程与结果并重的精细化管理。</p>

        <div class="table-responsive">

            <table>

                <thead><tr><th>区域</th><th>内容</th><th>作用</th></tr></thead>

                <tbody>

                    <tr><td>案件概况</td><td>未结清案件数、逾期总金额、已结清案件、结清率</td><td>展示存量风险与清收成果</td></tr>

                    <tr><td>分案与催收</td><td>承诺还款(PTP)、待分案、催收中、已分案总数</td><td>追踪催收流程进展，暴露案件分配与执行卡点</td></tr>

        <p><strong>催收人员工作台（我的工作台）：</strong>面向一线催收员的个人任务聚合看板，展示：今日紧急待办（应还款客户、PTP到期、超期未跟进案件、待分案新案件、待审批减免等）；催收关键指标（在案案件数、逾期案件数、逾期总金额、今日回款、今日催记数、有效PTP件数）；快捷入口（逾期案件列表、分案管理、减免管理、催记管理）。登录后即可明确“今天必须做什么”，减少思考成本。</p>

 

        <h4 id="sec4.4.3">4.4.3 正常待还款（售后模块）</h4>

        <p><strong>功能解释：</strong>支持按姓名/手机号、市场、咨询、跟进标签、跟进时间范围、下单日期范围进行多维度筛选，便于处理待还款客户。点击“填写记录”打开催记表单，记录沟通结果执行跟进动作。</p>

        <div class="img-placeholder">📋 [示意图：正常待还款列表]</div>

 

        <h4 id="sec4.4.4">4.4.4 逾期管理（售后/贷后模块）</h4>

        <p><strong>功能解释：</strong>通过分层展示逾期客户（按逾期天数区间），记录催收跟进、联动回访任务，实现对分期付款逾期风险的有效控制与清收。支持按逾期天数（1-30天、31-90天等）筛选，批量发送提醒，填写催记，标记承诺还款（PTP）等。</p>

       

        <h4 id="sec4.4.5">4.4.5 减免管理</h4>

        <p><strong>功能解释：</strong>针对逾期客户，催收员可在线提交减免罚息或部分本金的申请，经审批流程后生效。</p>

        const sections = document.querySelectorAll('[id^="sec"]');

        const navLinks = document.querySelectorAll('.nav-list a');

        function setActiveLink() {

            let current = '';

            const scrollPos = window.scrollY + 100; // 偏移

            sections.forEach(section => {

                const top = section.offsetTop;

                const bottom = top + section.offsetHeight;

                if(scrollPos >= top && scrollPos < bottom) {

                    current = section.getAttribute('id');

                }

            });

            navLinks.forEach(link => {

                link.classList.remove('active');

                const href = link.getAttribute('href');

                if(href && href.substring(1) === current) {

                    link.classList.add('active');

                }

            });

        }

        window.addEventListener('scroll', setActiveLink);

        window.addEventListener('load', setActiveLink);

    })();
